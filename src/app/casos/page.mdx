# Casos

Un **caso** en el ámbito legal es una disputa o controversia entre partes que se resuelve a través del sistema judicial, basado en hechos y argumentos legales específicos.
Cada caso se clasifica según su tipo, que puede ser administrativo, judicial, laboral, entre otros. Cada tipo de caso conlleva **procesos** específicos asociados y cada proceso tiene **procedimientos** que se deben llevar a cabo.{{ className: 'lead text-justify' }}

Un **procedimiento** puede entenderse como una guía o plantilla detallada que describe paso a paso cómo realizar una tarea específica de manera consistente y eficiente. Por ejemplo, en un caso laboral el proceso comúnmente conlleva ciertos procedimientos como la presentación de la demanda, notificación al empleador, entrevistas, etc.{{ className: 'lead text-justify' }}

Al asignar tareas a un caso, si éste pertenece a un tipo que tiene un proceso definido con procedimientos específicos, podemos convertir directamente los procedimientos asociados a dicho proceso a tareas con solo un botón. Además, usted tiene la opción de crear tareas adicionales que sean específicas y relevantes para ese caso en particular. {{ className: 'lead text-justify' }}

## Funcionalidades

Desde el módulo de casos podemos:

1. Ver estadísticas de los casos.
2. Crear un nuevo caso.
3. Desde el listado de casos podemos:
   - Acceder a una vista rápida para ver detalles del caso.
   - Editar un caso.
   - Agregar notas a un caso.
   - Eliminar un caso.
   - Registrar un gasto a un caso.

## Registrar apoderados de un caso

Los casos tienen tres posibles participantes: demandado, demandante y fiador. El último no es obligatorio al crear un caso. Un demandante, es decir aquella persona que demanda, puede tener un apoderado. Un **apoderado** es la persona que tiene la capacidad jurídica para actuar en nombre y por cuenta de otra. El apoderamiento debe realizarse mediante lo que se denomina un poder. Esta información es requerida para poder generar el poder del caso.{{ className: 'text-justify' }}

Actualmente se puede generar un solo tipo de poder. En el futuro estos poderes se generarán automáticamente.

Requerimientos para generar el poder del caso:

- Al menos un demandado que sea persona no jurídica.
- Al menos un demandante que sea persona jurídica.
- Apoderado del demandante que sea persona no jurídica. De este apoderado también se necesitan los datos de la relación con el demandante.

En caso de que, al momento de generar el poder del caso, alguno de estos datos no esté, se abrirá un modal que indicará los campos faltantes.{{ className: 'text-justify' }}
