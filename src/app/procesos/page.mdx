# Procesos

Un **proceso** se refiere a una serie estructurada y sistemática de acciones o pasos que se llevan a cabo de manera secuencial o simultánea para alcanzar un objetivo específico. Describen la serie de pasos que deben seguirse para tratar un caso en particular.{{className:'lead text-justify'}}

Cada caso del sistema se clasifica según su tipo, que puede ser administrativo, judicial, laboral, entre otros. Cada tipo de caso conlleva procesos específicos asociados y cada proceso tiene procedimientos que se deben llevar a cabo.{{className:'lead text-justify'}}

Un **procedimiento** puede entenderse como una guía o plantilla detallada que describe paso a paso cómo realizar una tarea específica de manera consistente y eficiente.
Por ejemplo, si tengo un caso laboral, el proceso comúnmente conlleva ciertos procedimientos como la presentación de la demanda, notificación al empleador, entrevistas, etc.
Al asignar tareas a un caso, si éste pertenece a un tipo que tiene un proceso definido con procedimientos específicos, podemos convertir directamente los procedimientos asociados a dicho proceso a tareas.{{className:'lead text-justify'}}

Un proceso puede tener **procesos hijos**, es decir procesos que se desprenden de él. Al crear las tareas desde el proceso en el caso, si marcamos la última tarea como culminada se podrá seleccionar el proceso hijo siguiente. Esta metodología permite crear distintos caminos a través de los procesos dependientes del caso en cuestión.{{className:'lead text-justify'}}

## Funcionalidades

Desde el módulo de procesos podemos:

1. Crear un nuevo proceso con sus correspondientes datos.
2. Editar un proceso.
3. Eliminar un proceso.
4. Agregar procedimientos al proceso.
5. Desde el listado de procedimientos podemos:

- Agregar un nuevo procedimiento.
- Editar un procedimiento.
- Eliminar el procedimiento.
- Cambiar el orden de los procedimientos.

6. Asignar procesos hijos.

Al clickear sobre el nombre del proceso, accedemos a la vista detallada de todos los procedimientos que conforman dicho proceso. Desde aquí podemos agregar nuevos procedimientos, editarlos, eliminarlos y cambiar su orden.

### Agregar procedimientos a un proceso

Desde el botón **agregar procedimiento** podemos crear un nuevo procedimiento incluyendo nombre, número de paso, descripción y dependencias correspondientes.
Las dependencias señalan qué otros procedimientos dentro del mismo proceso son necesarios para llevar a cabo dicho procedimiento.{{className: "text-justify"}}

Un procedimiento puede tener **acciones**. Las acciones posibles son: crear un gasto, mandar un request, enviar un email o llenar un campo.{{className: "text-justify"}}

A su vez, el procedimiento puede tener **recordatorios**. Un recordatorio es una herramienta o función diseñada para ayudar a las personas a recordar eventos importantes, tareas pendientes o cualquier otra información relevante en un momento específico o de manera periódica. Este recordatorio se envía en forma de notificación al usuario seleccionado en la fecha y horario especificado para recordarle acerca de este procedimiento.{{className: "text-justify"}}

### Cambiar el orden de los procedimientos

Para cambiar el orden de los procedimientos, se utiliza la función de arrastrar y soltar. Cada procedimiento está identificado por un número secuencial de pasos. Al agregar uno nuevo, se asigna automáticamente el siguiente número secuencial disponible después del último procedimiento existente. No se permite la duplicación de números de paso entre procedimientos.{{className: "text-justify"}}

## Procesos hijos

Un proceso puede tener **procesos hijos**, es decir procesos que se desprenden de él. ### Agregar un proceso hijo

Al crear un proceso, podemos seleccionar sus procesos hijos. En este listado aparecerán todos los procesos existentes. Un proceso puede tener ninguno, uno o más de un proceso hijo.{{className:'text-justify'}}

### Definir caminos entre procesos

Tenemos un caso de tipo corporativo, y creamos las tareas desde el proceso. Estas tres tareas pertenecen al proceso corporativo.{{className:'text-justify'}}

Al marcar la última tarea del proceso como culminado podemos seleccionar el proceso siguiente y se crearán las tareas correspondientes.{{className:'text-justify'}}
